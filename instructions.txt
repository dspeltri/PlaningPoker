AI Handover – Planning Poker Web App
1. Goal
Recreate a real‑time Planning Poker web app for agile estimation with:

A visual table in the center, participants arranged around it as seats.
Participants list on the left as tiles.
Backlog on the right (list of items to estimate).
Cards at the bottom, Reveal/Reset buttons under the table.
Name + avatar selection dialog on first load.
Real-time sync between users using Socket.IO.
Deployed on a Node‑friendly host (e.g., Render) as a single Node.js web service.
No external integrations (Azure DevOps, Vercel serverless) are needed in the final version. The last agreed scope is:

Simple Express + Socket.IO server.
Static HTML/JS/CSS front-end.
Backlog items are entered manually into the UI (paste from ADO or elsewhere).
Multi‑user real-time behavior via WebSockets.
Important: There were multiple iterations experimenting with Azure DevOps integration and Vercel; the final implementation removes that complexity and returns to a clean Node/Socket.IO + static front-end design.

4. Server Implementation (server.js)
The final server.js should:

Create an Express app.
Serve static files from public.
Create an HTTP server and attach Socket.IO.
Maintain rooms in memory.
Listen on process.env.PORT || 3000.
4.1 Target server.js content
js

Copy code
// server.js
// Simple Planning Poker server without external integrations

const express = require("express");
const http = require("http");
const { Server } = require("socket.io");

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: { origin: "*" },
});

// Serve static files
app.use(express.static("public"));

// In-memory room state: { roomId: { revealed: bool, votes: { name: value } } }
const rooms = {};

function getRoom(roomId) {
  if (!rooms[roomId]) {
    rooms[roomId] = { revealed: false, votes: {} };
  }
  return rooms[roomId];
}

io.on("connection", (socket) => {
  let currentRoom = null;
  let currentName = null;

  socket.on("joinRoom", ({ roomId, name }) => {
    if (currentRoom) socket.leave(currentRoom);

    currentRoom = roomId || "default-room";
    currentName = name || "Anonymous";

    socket.join(currentRoom);
    socket.emit("stateUpdate", getRoom(currentRoom));
  });

  socket.on("updateName", (name) => {
    currentName = name || "Anonymous";
  });

  socket.on("vote", (value) => {
    if (!currentRoom) return;
    const room = getRoom(currentRoom);
    room.votes[currentName || "Anonymous"] = value;
    io.to(currentRoom).emit("stateUpdate", room);
  });

  socket.on("reveal", () => {
    if (!currentRoom) return;
    const room = getRoom(currentRoom);
    room.revealed = true;
    io.to(currentRoom).emit("stateUpdate", room);
  });

  socket.on("reset", () => {
    if (!currentRoom) return;
    rooms[currentRoom] = { revealed: false, votes: {} };
    io.to(currentRoom).emit("stateUpdate", rooms[currentRoom]);
  });

  socket.on("disconnect", () => {
    // No further cleanup done in current implementation
  });
});

const PORT = process.env.PORT || 3000;
server.listen(PORT, () => {
  console.log(`Planning Poker running at http://localhost:${PORT}`);
});

5. Front-end Implementation (public/index.html)
5.1 Requirements & behavior
The front‑end should:

On load:

Show an overlay dialog asking:
Name (required, fallback “Anonymous”).
Avatar choice (from 6 images).
When the user presses Enter/Continue:
Hide the dialog.
Set nameInput at top.
Emit joinRoom with room ID derived from window.location.hash.
Emit updateName.
Maintain local state:

items: array of backlog item strings (entered manually).
currentItemIndex: index of current item.
Backlog:

Textarea itemsInput for “Items (one per line)”.
Button Add Items:
Splits by newline, trims, filters empties.
Stores into items.
Sets currentItemIndex = 0 if not empty.
Renders backlog tiles.
Previous and Next buttons:
Change currentItemIndex and call handleReset() to clear votes.
Backlog tiles:
Show “Item X of N”.
Click to select that item (and reset votes).
Participants:

Derived from currentState.votes coming from server.
Each tile shows:
Avatar image (from chosen avatar).
Name.
“Hidden” (grey) or actual value after Reveal.
Table:

Seats arranged in a circle around the center:
centerX = 210, centerY = 210, radius ~ 190.
Angle computed as 2π * index / count - π/2, so first starts at the top.
Each seat has avatar, name, and Hidden/value.
Cards:

Bottom row values:

Reveal/Reset:

9. Out-of-Scope / Removed Functionality
The following were explored during development but are not part of the final implementation:

Azure DevOps integration

Earlier we added:

/api/backlogByIds to fetch stories by ID.

This has been removed in the final version to keep the app simple and portable.
If re-introduced, must be done with:

Proper filtering by WorkItemType and AreaPath.
Vercel deployment

Tried to deploy the Node/Socket.IO server to Vercel.
Hit Cannot GET / and serverless/WebSocket limitations.
Final decision: use a Node‑friendly host (Render/Railway/Fly/Heroku) instead.
10. Known Limitations
In‑memory state only:
Restarting the server wipes all votes and backlog items.
Per‑client avatars not persisted server‑side:
All seats/tiles currently use the same avatar as the local user; upgrading to per‑user avatars requires extending server state.
No authentication / access control:
Anyone with the URL can join a room and vote.
If hosted publicly, consider adding a simple password or SSO in front.
